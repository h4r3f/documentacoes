Get started

Before we get started, you will need to choose a text editor to work on the configuration of the app. 

One of the best editors you could use is called Sublime text (for windows, mac or linux). You can download it by following this link.

Once you have installed sublime text on your computer, open it, go to File > Open. You will now need to open the configuration file, called api.json, located at www/api that came with the splendiz app.

All done ? great. now we can have some fun and make the app !

 

Build the app - the configuration file

Now if you have never seen a json file before, it might look frightening. But don't worry ! it is extremely straightforward and all you need to do with this file is enter the needed information. 

This is the file were you will be entering all the information needed to build the app. 

What you see if you take out the comments (the text preceded with "//") is something like this :

{

"menus" :{
  ... some stuff about the menu ...
},
"configuration" {
 ... some more stuff about the configuration ..
},
"informations" {
 ... some more stuff about the informations of the restaurant ...
}

}

these are the three main sections of the file.

The "menus" section is contained inside the brackets { } and is filled with information about the app menus, the prices and the sizes of every dish. 

the "configuration" section contains general information about the restaurant like the services offered.

the "informations" sections covers mostly the opening hours of the restaurant. 

We will slowly go trough the file starting with the menu section and you will be amazed at how easily you will have your app ready. 

menu section

Let's begin with the menu section of the file. 

the menu section contains 4 sub sections, corresponding to the tabs of the menu. Each tab will corresponds to a different type of food inside the final app. So if you look at what is allready written, you will see that there will be an appetizers tab, a menus tab,  a drinks tab and a desserts tab. Each tab section contains all of the images and names of the dishes that will be showed inside it. Meaning that the "appetizers" section will contain the informations about the appetizers the restaurant is offering

you can see something similar to this. 

"menus":
{
"appetizers" : {
 ... information about the appetizers tab ...
},
"menus" : {
 ... information about the menu tab ...
},
"drinks" : {
 ... information about the drinks tab ...
}
"desserts" : {
 ... information about the desserts tab ...
}
}

Now you will change the values inside that section to have your own tabs and items inside your app. Of course you can choose the tabs and their content. Fore example, if you want to have a "pizza" tab, a "ice creams" tab and a "desserts" tab, you will have a structure like this one :

"menus":
{
"pizza" : {
 ... information about the pizza tab ...
},
"ice creams" : {
 ... information about the ice creams tab ...
},
"desserts" : {
 ... information about the desserts tab ...
}
}

The structure inside each tab section will be the same. So lets go trough one tab section and you will know how to do it for every tab.

inside each tab, you have four values. "use", "name", "icon" and a sub section called "dishes". Let's go trough each one : 

"use" indicates if the tab will be showed inside the app or not. if you want to use it inside, put "true" inside of the value. if not, put "false"

"use" : "true"  ----> the tab will be used inside the app
"use" : "false" ----> the tab will not be used inside the app

 

"name" is the name of the tab. Usualy you can put the same name for the "name" proprety and the general tab name. 

 

"icon" corresponds to the icon that will be displayed in the tab bar for the current tab. You can take a llok at all the available icons inside the Splendiz/www/img/food_icons folder. For example, if you want to use the tab showing a slice of pizza (the image 117.png), the put the following proprety for the "icon" value :

"icon" : "img/food_icons/117.png"

 

The "dishes" section is a little bit longer. it contains a section for every dish of the app. You can give a number name for everyone of these sections. The real name of each dish will be informed inside the section.

"one" {
    ... information about the first dish of the tab ...
},
"two" {
    ... information about the second dish of the tab ...
},
"three" {
    ... information about the third dish of the tab ...
},
"four" {
   ... information about the fourth dish of the tab ...
},
"five" {
   ... information about the fifth dish of the tab ...
}

Now let's look inside the dish section. You can see the values "name", "imageUrl" and "ingredients" and a sub section called "sizes"

the "name" value indicates the name of the dish, the "imageUrl" is the link to the image of the dish and "ingredients" contains the description of the ingredients of the dish. As an example, if you have an dish called "my amazing pizza", with the url of the image of "http://www.pizza.com" and the ingredients "tomatoes, chicken, spices", the first three values of the dish information would look like this : 

"one" : {
   "name" : "my amazing pizza",
   "imageUrl" : "http://www.pizza.com",
   "ingredients" : "tomatoes, chicken, spices"
}

 

One important note is that you don't have to have to enter a url of the image available on the internet. If you simply have the image on your computer, put it inside the Splendiz/www/api/images folder and indicate the path to your image inside the "imageUrl" value. Let's take an example. You have an image on your computer called "pizzaImage.png". You will have to put that image inside the Splendiz/www/api/images folder and then write 

"imageUrl" : "api/images/pizzaImage.png"

 Putting the images directly in the folder will make the app faster since it won't have to load the images from the internet.

The "sizes" section contains the different sizes of the dish, with the price and the quantity for each size. Let's take some examples. If you want to have three different sizes for your item with different prices in each size, enter something like this :

"sizes" : {
  "small" : {
      "price" : "10",
      "quantity" : "small"
    },
  "medium" : {
      "price" : "15",
      "quantity" : "medium"
    },
  "big" : {
      "price" : "20",
      "quantity" : "big"
  }
}

If on the other hand, you desire to have only one size, without showing the quantity, enter something like this :

"sizes" : {
  "" : {
      "price" : "10",
      "quantity" : ""
    }
}

Note how we left the name of the size and the quantity empty.

Now that you know what to enter, you can input the needed informations for each tab section.

configuration section

Now that you are done with the menu, we can move on to the configuration section. 

"configuration" : {
"options": {
    ... options ...
}
}

you can see that it is composed of only one sub sections called "options", comosed of three other subsections called "takeout", "eatin" and "delivery".

"options" : {
      "takeout" : {
        "name" : "Take out",
        "stateName" : "takeout",
        "url" : "templates/takeout.html",
        "icon" : "ion-bag"
      },
      "eatin" : {
        "name" : "Eat in",
        "stateName" : "eatin",
        "url" : "templates/eatin.html",
        "icon" : "ion-android-restaurant"
      },
      "delivery" : {
        "name" : "Delivery",
        "stateName" : "delivery",
        "url" : "templates/delivery.html",
        "icon" : "ion-home"
      }
}

Each of these sections stands for a service the restaurant is offering. The only thing that you can change in here is the name of the Service. If for example you want the delivery service to be called "we bring it to you", change the "name" value inside the "delivery" section to that : 

"delivery" : {
"name" : "we bring it to you",
...
...
}

if your restaurant doesn't offer one of these options, you can simply remove the section from the "options" section. If for example, you do not offer delivery, remove the delivery section and the last comma before the delivery section.

"options" : {
      "takeout" : {
        "name" : "Take out",
        "stateName" : "takeout",
        "url" : "templates/takeout.html",
        "icon" : "ion-bag"
      },
      "eatin" : {
        "name" : "Eat in",
        "stateName" : "eatin",
        "url" : "templates/eatin.html",
        "icon" : "ion-android-restaurant"
      }  <-- the comma here has been removed
}

 

informations section

We can now get started on the third and final sections, called "informations". This section contains the cost of the delivey, contact informations about the restaurant and the opening hours. 

"informations"{
"delivery_cost" : "the delivery cost...",
"contact_informations" : {
    ... the contact informations ...
},
"opening_hours" : {
    ... the opening hours of the restaurant ...
}
}

the "delivery cost" value contains only a number, which represents the cost of the delivery. It can be any price, and should be equal to "0" if the delivery is free. If your restaurant doesn't offer delivery, you can ignore this value and just leave it. 

 

the "contact_informations" sub section contains the adress, the geolocalisation, the phone number and the email of the restaurant. The "adress" sub section contains two values called "first_part" and "second_part". put the street number and the street name inside the "first_part" and the postal code and city name inside the "second_part".

"contact_informations" : {
"adress" : {
  "first_part" : "first part of the adress",
  "second_part" : "second part of the adress"
},
"geolocalisation" : {
  "latitude" : "42.77722199999999",
  "longitude" : "3.0870250000000397"
},
"phone" : "0123456789",
"email" : "email@example.com"
},

 

Now for the "opening_hours" section, it contains 7 sub sections called "1" to "7". Each number represents the day of the week and contains the values "open","first_part","second"part".

"1" : {
"open" : "true or false",
"first_part" : {
  ... inormations about the first part of the day ...
},
"second_part" : {
  ... informations about the second part of the day ...
}
}

the first value is "open" and contains the value "true" if the restaurant is open on that day, and the value "false" if it is not.

the "first_part" and the "second_part" sections are identical and contain the following structure : 

{
"time" : "",
"max_eating_time" : "",
"max_takeout_time" : "",
"max_delivery_time" : "",
}

the "time" value should contain the opening and closing hours of that part of the day, in the 24 hours format, separated by "-". If you are open from 6am to 11am, you shoul put :

"time" : "6-11",

the "max_eating_time", "max_takeout_time" and "max_delivery_time" represent the maximum hour of each service for that part of the day.

if your restaurant doesn't close during the day, simply put the opening hour of the second part equal to the closing part of the first.

if you are closed one day, also put all the values equal to 99. As an exmaple, if you are closed on sundays, simply put :

 "7" : {
  "open" : "false",
  "first_part" : {
    "time" : "99-99",
    "max_eatin_time" : "99",
    "max_takeout_time" : "99",
    "max_delivery_time" : "99"
    },
  "second_part" : {
    "time" : "99-99",
    "max_eatin_time" : "99",
    "max_takeout_time" : "99",
    "max_delivery_time" : "99"
    }
}

icons and app name

Let's start by setting the icon and splash screen of the app.

First, make sure that you should have a 512*512px  png image of your icon and a 2048*2048px png image of your splash screen (The splash screen is the first screen that pops up when the app is launched)

To automatically create the variety off different sizes of your icon and splash screen needed by the app. You can go to this website. There, you can upload your images and you will get a folder containing all the files that you need.

Make sure that you name your icon image "icon.png" and your splash image "splash.png" before you upload it to the website

 

Once you have that folder from the website, uncompress it and open it. The icon images for ios should be in res/icon/ios and the icon images for android in res/icon/android. In the same way, the splash images for ios are in res/screen/ios and the ones for android in res/screen/android. You will now need to copy and paste those images in the corresponding folder of your app.

  • put the images from res/icon/ios in the icons/ios folder of the app.
  • put the images from res/icon/android in the icons/android folder of the app.
  • put the images from res/screens/ios in the splash/ios folder of the app.
  • put the images from res/screens/android in the splash/android folder of the app.

There you go. The icons of your app are ready. Now the last step is to change the app name. open the file called config.xml in the app folder and change the app name on the line 3.

<name>put your app name here</name>

Finaly, put the 512*512 icon image you created at the begining inside the www/img folder. This one will be used inside of your app and will be displayed in the side menu.

Uploading the app

Now that your app is ready, how do you publish it ? This app is based on a framework called ionic. This means that you can use the tools that they have made to publish your app very easily. For more informations, go to there website.

But there is an easier way to do it. You can build your app for android and ios directly on this site. You will need a google developer and an apple developer account. The site will output the source code of your ios and android app. The last step will ten be to publish the android app on google play and itunes store.
